# Design Engineer Course - Master Plan & Progress Tracker

> **Last Updated:** December 2025

## Overview

A comprehensive, "Choose Your Own Adventure" style course for becoming a Design Engineer. This course supports two primary tracks across three platforms, plus a convergence track for those pursuing mastery of both.

### Tracks

- **Design for Engineers** - Designers learning to build
- **Engineering for Designers** - Engineers learning to design

### Platforms

- **Web** (HTML/CSS/JavaScript)
- **iOS** (Swift/SwiftUI)
- **Android** (Kotlin/Jetpack Compose)

### Convergence

Where design and engineering skills merge into true Design Engineering mastery.

## Content Guidelines

### Tone

- Conversational but professional
- Assume intelligence, explain concepts
- "We" language (learning together)
- Real-world examples over theoretical
- Link concepts back to the DE role

### Lesson Template

```markdown
# [Lesson Title]

## [Main Content Sections]

Link to next lesson or related content
```

### Code Examples

- Keep examples minimal and focused
- Show the "why" not just the "how"
- Provide working examples that can be copied
- Comments only for non-obvious things

## User Notes System

### Overview

Logged-in users can take notes as they progress through the course. Notes are stored in Cosmic and associated with their Clerk user ID.

### Features

- **Floating Notes Panel:** Collapsible panel accessible from any lesson
- **Lesson-Specific Notes:** Notes tied to the current lesson for contextual learning
- **Course-Wide Notes:** General notes section accessible from user dashboard
- **Auto-Save:** Notes save automatically as users type
- **Markdown Support:** Rich text formatting with markdown
- **Exercise Responses:** Special note type for "Try It Yourself" exercises

### Cosmic Object Types

#### `course-notes`

```json
{
  "title": "course-notes",
  "slug": "course-notes",
  "metafields": [
    { "key": "user_id", "type": "text", "required": true },
    { "key": "lesson_path", "type": "text", "required": true },
    { "key": "note_type", "type": "select", "options": ["general", "exercise", "highlight"], "required": true },
    { "key": "content", "type": "markdown", "required": true },
    { "key": "is_pinned", "type": "switch", "default": false }
  ]
}
```

#### `course-progress`

```json
{
  "title": "course-progress",

    { "key": "user_id", "type": "text", "required": true },
```

### UI Components

### Access Control Logic

```typescript
const FREE_MODULES = {
  'design-track/web': ['01-foundations'],
  'design-track/ios': ['01-hig-fundamentals'],
  'design-track/android': ['01-material-design'],
  'engineering-track/web': ['01-html-fundamentals'],
  'engineering-track/ios': ['01-swift-basics'],
  'engineering-track/android': ['01-kotlin-basics'],
  '00-introduction': ['*'], // All intro lessons free
};

function canAccessLesson(userEnrollment, lessonPath) {
  // Check if lesson is in free tier
  if (isFreeTierLesson(lessonPath)) return true;
  
  // Check enrollment level
  if (userEnrollment.access_level === 'full') return true;
  
  // Check specific track access
  return hasTrackAccess(userEnrollment.access_level, lessonPath);
}
```

## API Routes

### Notes API

- `GET /api/course/notes` - Get user's notes (optional: by lesson)
- `POST /api/course/notes` - Create a note
- `PATCH /api/course/notes/[id]` - Update a note
- `DELETE /api/course/notes/[id]` - Delete a note

### Progress API

- `GET /api/course/progress` - Get user's progress
- `POST /api/course/progress` - Mark lesson progress
- `GET /api/course/progress/stats` - Get completion statistics

### Enrollment API

- `GET /api/course/enrollment` - Get user's enrollment status
- `POST /api/course/enrollment/check` - Check access for specific lesson

### Payment API

- `POST /api/course/checkout` - Create LemonSqueezy checkout session
- `POST /api/webhooks/lemonsqueezy` - Handle payment webhooks
